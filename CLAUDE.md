# 📋 프로젝트 기술 정리 지침 (포트폴리오/이력서용)

## 🎯 목적
이 프로젝트의 기술 스택과 핵심 내용을 분석하여,
**포트폴리오 및 이력서에 쓸 수 있는 문서**를 생성한다.

---

## 📁 1단계: 프로젝트 구조 파악

아래 순서대로 코드베이스를 탐색해라.

```
1. 루트 디렉토리 구조 확인 (ls -la, tree 등)
2. README가 있으면 먼저 읽기
3. requirements.txt / pyproject.toml / poetry.lock 확인 → 의존성 전부 파악
4. 핵심 디렉토리 (src/, app/, api/, models/, services/ 등) 탐색
5. 메인 진입점 파일 (main.py, app.py 등) 읽기
6. 라우터/컨트롤러, 서비스 레이어, DB 모델 순으로 읽기
7. .env.example 또는 config 파일로 인프라 구성 파악
```

---

## 🔍 2단계: 분석해야 할 항목

### A. 기술 스택 추출
- **언어**: Python 버전
- **프레임워크**: FastAPI, Flask 등 + 버전
- **AI/ML 관련**: 사용한 모델(OpenAI, Claude 등), 감정분석 라이브러리(transformers, VADER, KoBERT 등)
- **시 생성 관련**: LLM API, 프롬프트 엔지니어링 방식
- **DB**: PostgreSQL, MySQL, SQLite, MongoDB, Redis 등
- **ORM**: SQLAlchemy, Tortoise-ORM 등
- **인프라/배포**: Docker, AWS, GCP, Nginx 등
- **기타**: Celery, WebSocket, OAuth 등

### B. 핵심 기능 파악
- 사용자 입력 → 감정 분석 파이프라인이 어떻게 구성되어 있는지
- 감정 분석 결과가 시 생성에 어떻게 연결되는지 (프롬프트 구조 확인)
- API 엔드포인트 목록 및 역할
- 팀원 간 역할 분담 흔적 (git blame, 디렉토리 구조, 주석 등으로 유추)

### C. 아키텍처 파악
- 전체 서비스 흐름 (입력 → 처리 → 출력)
- 레이어 구조 (Router → Service → Repository 등)
- 외부 API 연동 구조

## ⚠️ 주의사항

- **모르는 부분은 `TODO: 직접 확인 필요`로 표시**하고 넘어가라. 추측으로 채우지 마라.
- 버전 정보는 requirements.txt 등에서 **정확히** 가져와라.
- 감정 분석 모델과 시 생성 LLM이 **각각 무엇인지** 명확히 구분해서 적어라.
- 팀 프로젝트이므로 **본인 담당 부분**을 특정할 수 있으면 별도로 표시해라 (`[내 담당]` 태그 등).
- 분석 완료 후 요약을 먼저 나에게 보고하고, 확인 후 파일 생성해라.
